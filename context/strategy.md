# X 運用戦略

## ツイートタイプ別設計

### 1. 速報ツイート（Claude/Anthropic 新機能リリース時）

**目的**: トレンドに乗る・リーチ拡大
**頻度**: Claude 新機能リリース時（不定期）
**フォーマット**:
- 1行目: 「〇〇が出た」「〇〇リリースされた」（事実ファースト）
- 2行目: 「自分的に一番刺さったのは〜」（個人の視点）
- 3行目: 詳しく書いた → note リンク
**ハッシュタグ**: #ClaudeCode か #Claude のどちらか1つだけ

### 2. 実況ツイート（日常・作業中）

**目的**: 親近感・継続フォロー動機
**頻度**: 週 3〜4 本
**フォーマット**:
- 今やってること + 結果（Before/After）
- 「〇〇してたら〇〇になった」形式
- 1ツイートで完結
**ハッシュタグ**: なし or #ClaudeCode のみ

### 3. ノウハウスレッド（テクニック・Tips）

**目的**: 保存・リポスト・フォロワー獲得
**頻度**: 週 1〜2 本
**フォーマット**:
- 1ツイート目: フック（「Claude Code でこれやってる人少なすぎる」等）
- 2〜7ツイート目: 具体的な手順・Tips（1ツイート1アイデア）
- 最終ツイート: まとめ + note 誘導 or フォロー訴求
**ハッシュタグ**: スレッドの最後のみ 1〜2 個

### 4. 実績報告ツイート（スキ数・PV・案件）

**目的**: 信頼構築・実績の見える化
**頻度**: 月 1〜2 本
**フォーマット**:
- 数字 + 一言コメント（「〇〇記事がスキ16になってた」「〇〇円の案件受注した」）
- 自慢ではなく「こうやったらこうなった」報告スタイル
**ハッシュタグ**: なし

### 5. note 誘導ツイート（記事公開時）

**目的**: note→X→note の循環を作る
**頻度**: note 記事公開のたび
**フォーマット**:
- 記事の一番おいしいところを 1〜2 行で
- 「続きは note に書いた」+ リンク
- フックが先、リンクは後

## 投稿タイミング

| ツイートタイプ | 推奨時間帯 |
|---|---|
| 速報 | リリース当日の朝 7〜9 時 |
| 実況 | 平日 12 時〜13 時 or 21〜22 時 |
| ノウハウスレッド | 火〜木の 20〜21 時 |
| 実績報告 | 週末の昼 |
| note 誘導 | note 公開と同タイミング or 1〜2 時間後 |

## エンゲージメント戦略

- Claude Code 界隈のツイートにリプライ（目立つよりも「価値ある返信」を意識）
- 他の Claude Code 系インフルエンサーのツイートを引用 RT（自分の視点を加える）
- リプライで「それどうやりましたか？」が来たら詳細 note に誘導

## コンテンツ循環モデル

```
note記事公開
    ↓
X に速報ツイート or note 誘導ツイート
    ↓
X でバズったネタ / リプライの質問 → note のネタ帳に追加
    ↓
ノウハウスレッド化 → さらに拡散
    ↓
スレッドを深掘りした有料 note 記事
```

## やらないこと

- ハッシュタグ大量使用（スパムに見える）
- 「フォローお願いします」的な直接的な訴求
- 宣伝色が強い文言（「〇〇を使ってみて！」等）
- 他ツール（ChatGPT 等）の悪口
- 嘘の数字・盛った実績

## PDCA

- 週次: エンゲージメント率（いいね/インプレッション）を確認
- 月次: フォロワー増加数・note への流入数を確認
- `context/performance.md` に記録して傾向を分析
