# x-pipeline

## プロジェクト概要

moyuchi の X（Twitter）運用パイプライン。note-pipeline で生成した記事を X 向けに展開し、Claude / Claude Code 特化アカウントとしてフォロワーを増やす。

## アカウント設計

- **ハンドル**: @moyuchi_cc（推奨）または @moyuchi_claude
- **表示名**: moyuchi | Claude Code実況
- **プロフィール文**:
  ```
  情報系大学生 ／ Claude Codeで案件受注→初月10万達成 ／
  新機能速報×実録×稼ぎ方を全公開 ／「Claudeのことならここ」を目指してる ／
  note更新中📝
  ```
- **note リンク**: https://note.com/moyuchi_aistu
- **ピン留め候補**: 「大学生がClaude Codeでクラウドワークス案件を受注→納品してみた全記録」

## ポジショニング

- ジャンル: **Claude / Claude Code 特化** × 副業 × 大学生
- 差別化: 「実際に稼いでいる大学生の一次情報」をリアルタイムで流すアカウント
- 目標フォロワー: 3ヶ月で1,000人

## ツイートタイプ

| タイプ | 頻度 | 役割 |
|---|---|---|
| 速報ツイート | Claude新機能リリース時 | トレンド乗り・リーチ拡大 |
| 実況ツイート | 週3〜4本 | 親近感・継続フォロー動機 |
| ノウハウスレッド | 週1〜2本 | 保存・拡散 |
| 実績報告 | 月1〜2本 | 信頼構築 |
| note誘導 | 記事公開時 | note→X→note の循環 |

## Notion 連携

note-pipeline と Notion をハブとして双方向に連携する。

### 共有 Notion DB

| DB | data_source | 用途 |
|---|---|---|
| note記事管理 | `collection://812aa728-8d3e-42e4-a9cd-6a91c303b2c2` | X未投稿記事の検知・ステータス管理 |
| ネタ帳 | `collection://1a603b4f-d1e4-4ed7-8c75-c7c0a5b7e595` | X発案ネタの note-pipeline への引き渡し |

### note記事DB の追加フィールド（x-pipeline 用）

| フィールド名 | タイプ | 値の選択肢 |
|---|---|---|
| `Xステータス` | select | 未投稿 / 草稿完成 / 投稿済み |
| `X草稿パス` | text | ドラフトファイルの絶対パス |
| `XURL` | url | 投稿後の X ポスト URL |

### 連携フロー

**note → X:**
1. `/note-run` が記事を完成させる
2. Notion の `Xステータス` を **未投稿** にセット（自動）
3. `/x-run` 起動時に未投稿記事を自動検知 → ツイート生成
4. 投稿後に `Xステータス` を **投稿済み** に更新

**X → note:**
1. `/x-run trend` でトレンドをスキャン
2. 「記事化すべき」ネタを Notion ネタ帳に **X発案** として追加（自動）
3. `/note-run` がネタ帳の X発案ネタを優先ピックアップ

## スキル

- `/x-run` : Notion の未投稿記事を拾いツイート生成。またはトレンドスキャン・単独ツイート生成
- `content-engine` : 素材からプラットフォーム別コンテンツ変換（affaan-m より導入）
- `crosspost` : X + 他プラットフォーム同時展開（affaan-m より導入）

## フォルダ構成

- `context/` : プロフィール・戦略・文体・パフォーマンスデータ
- `drafts/threads/` : スレッドツイート草稿
- `drafts/singles/` : 単発ツイート草稿
- `published/` : 投稿済みログ
- `.claude/skills/` : スキル定義

## ルール

- 文体: note と同じ地声。ただし X 用に圧縮・フック重視
- AI 感のある言い回し禁止（「〜と言えるでしょう」等）
- ハッシュタグは 0〜2 個まで（スパムしない）
- 宣伝感を出さない。体験・実況が軸
- 送信はユーザーが行う。スキルはドラフトを生成するだけ

## note-pipeline との連携

- note 記事公開 → `/x-run from-article {ドラフトパス}` で X 用コンテンツ自動生成
- X でバズったネタ → note のネタ帳に追加
