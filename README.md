# x-pipeline

moyuchi の X（Twitter）自動運用パイプライン。note-pipeline と Notion をハブに連携し、投稿草稿の生成からステータス管理までを自動化する。

---

## 使い方

```bash
/x-run                              # Notion の未投稿記事一覧を表示して選択
/x-run from-article {ドラフトパス}   # note 記事を直接指定してツイート生成
/x-run standalone                   # トピックを指定して単発/スレッド生成
/x-run trend                        # Claude 最新情報をスキャンしてツイートアイデア生成
/x-run done {notion_id} {x_url}     # 投稿完了後の Notion ステータス更新
```

## ツイートタイプ

| タイプ | 頻度 | 目的 |
|--------|------|------|
| 速報 | 新機能リリース時 | トレンド乗り・リーチ拡大 |
| 実況 | 週3〜4本 | 親近感・継続フォロー動機 |
| ノウハウスレッド | 週1〜2本 | 保存・拡散 |
| 問いかけ型 | 週1〜2本 | リプライ数増・エンゲージメント |
| 図解カード型 | 週1本 | ブックマーク・保存率向上 |
| 実績報告 | 月1〜2本 | 信頼構築 |
| note 誘導 | 記事公開時 | note↔X 循環 |

---

## アルゴリズム対策（重要）

| ルール | 理由 |
|--------|------|
| **外部リンクはリプライに置く** | 本文直埋めでリーチ30〜90%低下 |
| **画像または動画を必ず添付** | テキストのみの約2倍のスコア |
| **投稿後30分は初動エンゲージを確保** | 10〜20件が拡散の閾値 |
| **スレッドは自分でリプライして繋げる** | アルゴリズム最大75点ボーナス |
| **火〜木の朝9〜11時・夜19〜21時** | エンゲージメントが最も高い時間帯 |

## 草稿ファイルのフォーマット

```markdown
---
type: thread | single
source_article: {note記事パス or "standalone"}
created: YYYY-MM-DD
status: draft
note_url: {URL or TBD}
---

## ツイート1
{本文}

## ツイート2
{本文}

## 投稿メモ
- 推奨投稿時間: {時間帯}
- ハッシュタグ案: {0〜2個}
```

---

## ルール

- **投稿はユーザーが行う。スキルは草稿を生成するだけ。**
- ブラウザ操作時は `@moyuchi_cc` でログイン中であることを先に確認する
- 外部リンクは本文ではなく投稿後のリプライに置く
- ハッシュタグは 0〜2 個まで（スレッドは最後のツイートのみ）
- AI 感のある言い回し禁止
